# private 생성자나 열거 타입으로 싱글턴임을 보증하라

싱글턴이란? 인스턴스를 오직 하나만 생성할 수 있는 클래스

## 방법 1 (public static final 필드 방식)

### 장점

- 간결함
- 싱글턴임이 API에 명백히 드러남

### 단점

- 리플렉션으로 private 생성자 호출 가능
- 직렬화 시 추가 작업 필요

## 방법 2 (정적 팩터리 메서드 방식)

### 장점

- API 변경 없이 싱글턴이 아니게 변경 가능
- 정적 팩터리를 제네릭 싱글턴 팩터리로 만들 수 있음
- 메서드 참조를 공급자로 사용 가능 (`Elvis::getInstance` → `Supplier<Elvis>`)

### 단점

- 직렬화 시 추가 작업 필요

## 방법 3 (열거 타입 방식)

### 장점

- 간결함
- 추가 노력 없이 직렬화 가능
- 리플렉션 공격에도 안전
- 대부분의 상황에서 가장 좋은 방법

### 단점

- Enum 외의 클래스를 상속할 수 없음 (인터페이스 구현은 가능)

### 역직렬화의 동작 원리

- 생성자를 호출하지 않고 객체를 만듦
- 파일에서 읽은 데이터로 메모리에 새 객체를 복원
- 기존 싱글턴 인스턴스와는 완전히 별개의 객체
