# Effective Java Item 69 — 예외는 진짜 예외 상황에만 사용하라

## 한 줄 요약
예외는 예외적인 상황을 알리는 수단이다. **흐름 제어**로 쓰지 마라.

## 왜 안 좋은가
- 예외는 **비정상 상황**을 표현하는 메커니즘이다.
- 정상 흐름에 예외를 쓰면 **가독성**이 떨어지고 **디버깅**이 어려워진다.
- JVM은 예외 경로에 최적화되어 있지 않아 **성능**에도 불리하다.

## 잘못된 예: 예외로 반복문 종료
```java
try {
    for (int i = 0; ; i++) {
        process(i);
    }
} catch (IndexOutOfBoundsException e) {
    // 반복 종료
}
```

## 권장 예: 조건으로 제어
```java
for (int i = 0; i < size; i++) {
    process(i);
}
```

## 원칙
- **예외는 진짜 예외**에만 사용하라.
- 루프 종료, 상태 분기 등 **정상적인 제어 흐름**은 조건문으로 처리하라.
- API 설계 시 **정상 상황**은 반환값/상태로 표현하고, **예외 상황**만 예외로 표현하라.

## 핵심 정리
- 예외는 오류 신호다. 제어 흐름 수단이 아니다.
- 코드를 읽는 사람이 “정상/비정상”을 바로 구분할 수 있게 하라.
