# Effective Java Item 51 — 메서드 시그니처를 신중히 설계하라

## 한 줄 요약

메서드 이름은 표준 규칙에 맞게 신중히 짓고, 매개변수는 짧고 명확하게 유지하라.

## 메서드 이름

- 표준 명명 규칙(아이템 68)을 따르고, 커뮤니티에서 널리 통용되는 이름을 선택한다.
    - 동사로 시작(get, set, find ..)
    - boolean 반환은 is/has/can

## 편의 메서드 남발 금지

- 메서드가 많아지면 문서화·테스트·사용이 어려워진다.
- 확신이 없으면 추가하지 말자.

## 매개변수 목록은 짧게

- 일반적으로 4개 이하가 좋다.
- 같은 타입의 매개변수가 연속으로 나오면 실수 위험이 커진다.

## 매개변수 줄이는 방법

1. 메서드를 쪼개기

- 큰 메서드를 직교적인 작은 메서드로 분리한다.
- 예: `subList()`로 범위를 좁히고 `indexOf()`로 위치를 찾는다.

2. 도우미 클래스(정적 멤버 클래스) 사용

- 여러 매개변수를 하나의 값 객체로 묶는다.
- 가독성과 내부 구현이 깔끔해진다.

3. 빌더 패턴 응용

- 일부 매개변수를 생략해도 될 때 유용하다.
- 도우미 클래스 + 빌더 조합도 가능하다.

## 타입 선택

- 매개변수 타입은 **클래스보다 인터페이스**가 낫다. (아이템 64)
    - 구현체가 변경되어도 인터페이스는 그대로 유지될 수 있다.

## boolean 대신 열거 타입

- `boolean`은 의미가 모호하다.
- 2개짜리 열거 타입이 의도를 명확히 한다.
- 예: `Thermometer.newInstance(true)`보다 `Thermometer.newInstance(TemperatureScale.CELSIUS)`가 명확하다.
