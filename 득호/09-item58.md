# 전통적인 for문 보다는 for-each 문을 사용하라

```java
for (Iterator i = c.iterator(); i.hasNext(); ) {
    E e = i.next();
    //...
}
```
또는

```java
for (int i = 0; i < a.length; i++) {
    //...
}
```

위 코드는 전통적인 for문을 사용한 예시이다.
하지만 반복자와 인덱스 변수는 모두 코드를 지저분하게 할 뿐이다.
실제로 필요한건 원소들 뿐이다.

인덱스는 네번이나 등장하고 혹시라도 잘못된 변수를 사용 했을 때 컴파일러가 잡아주지 못하는 실수를 할 수 있다.
나또한 알고리즘 문제를 풀변서 인덱스 변수 i를 j로 잘못 써서 틀린적이 자주 있다.

### 이러한 문제를 for-each 문을 사용해 해결할 수 있다.

`for-each` 문은 향상된 for 문이라고도 불린다.

```java
for (E e : c) {
    //...
}
```

반복 대상이 컬렉션이든 배열이든 상관없이, for-each 문을 사용해도 속도는 그대로다.
또한, 컬렉션을 중첩해 순회해야 한다면 for-each 문의 이점이 더욱 커진다.

```java
// 버그를 찾아보자
enum Suit { CLUBS, DIAMONDS, HEARTS, SPADES }
enum Rank { ACE, DEUCE, THREE, FOUR, FIVE, SIX}

static Collection<Suit> suit = Arrays.asList(Suit.values());
static Collection<Rank> rank = Arrays.asList(Rank.values());

List<Card> deck = new ArrayList<>();
for (Iterator<Suit> i = suit.iteraor(); i.hasNext(); ) {
    for (Iterator<Rank> j = rank.iterator(); j.hasNext(); ) {
        deck.add(new Card(i.next(), j.next()));
    }
}
```

위 코드는 버그가 발생하는 코드다. 
`i.next()`와 `j.next()`가 각각 반복문에서 한 번씩 호출되기 때문에
`i.next()`의 경우 내부 `for`문이 아닌 외부에서 사용되는 것을 의도했지만
의도와는 다르게 사용했고 이러한 코드는 문제를 찾는데 오랜 시간이 걸릴 수 있다.

또한 동일한 케이스로 이중 for문을 사용해 모든 경우의 수를 만드는 코드에서도 버그가 발생할 수 있다.

### for-each 문을 사용하면 이러한 문제를 방지할 수 있다.

```java
for (Suit s : suit) {
    for (Rank r : rank) {
        deck.add(new Card(s, r));
    }
```

위와 같이 향상된 For문을 사용하면 반복자나 인덱스 변수를 사용할 필요가 없기 때문에 코드가 훨씬 간결해지고, 버그가 발생할 가능성도 줄어든다.

### For-each를 사용하지 못하는 세가지 경우

1. 컬렉션을 순회하면서 선택된 원소를 제거해야 하는경우
    - 하지만 자바 8 이후부터는 `removeIf` 메서드를 사용해서 이 문제를 해결할 수 있다.
2. 리스트나 배열을 순회하면서 그 원소의 값 일부 혹은 전체를 변경하는 경우 인덱스를 사용해야한다.
3. 병렬 반복 : 여러 컬렉션을 병렬로 순회해야 하는 경우 반복자와 인덱스를 사용해야 한다.

### 결론
for - each 문 사용을 항상 고려하고
만약 사용할 수 없는 경우 실수를 조심하자.
