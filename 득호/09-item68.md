# 일반적으로 통용되는 명명 규칙을 따르라
자바 플랫폼은 명명 규칙이 잘 정립되어 있다.
덕분에 자바 개발자들은 클래스, 메서드, 변수 이름을 보고 그 의미를 쉽게 이해할 수 있다.

이미 널리 통용되는 명명 규칙을 따르지 않는 경우 이해하고 유지보수하기 어렵다.

자바 진영에서 사용되고 있는 규칙들을 알아보자.

**크게 철자 규칙과 문법 규칙으로 나뉜다.**

## 철자 규칙 (반드시 지켜야 하는 기본 형태)
1. 패키지와 모듈 이름은 각 요소를 . 으로 구분하여 계층적으로 짓는다.
   - 이때 요소들은 모두 소문자 알파벳 혹은 드물게 숫자로 이뤄진다.
   - 조직 외부에서도 사용될 패키지라면 인터넷 도메인 이름을 역순으로 사용한다 (예: `com.google`).
   - 각 요소는 8자 이하의 짧은 단어나 의미가 통하는 약어를 추천한다.

2. 클래스와 인터페이스 이름은 하나 이상의 단어로 이뤄지며, 각 단어는 대문자로 시작한다 (예: `FutureTask`).
   - max, min처럼 널리 통용되는 줄임말을 제외하고는 단어를 함부로 줄여 쓰지 않는다.
   
3. 메서드와 필드 이름은 첫 글자를 소문자로 쓴다는 점만 빼면 클래스 명명 규칙과 같다 (예: `ensureCapacity`).

4. 상수 필드를 구성하는 단어는 모두 대문자로 쓰며 단어 사이는 밑줄(_)로 구분한다 (예: `NEGATIVE_INFINITY`).
    - 상수 필드는 값이 불변인 static final 필드를 의미한다.
    
5. 지역변수는 다른 멤버와 비슷한 명명 규칙을 따르되, 문맥에서 의미를 쉽게 유추할 수 있다면 약어를 적극적으로 써도 좋다.

6. 타입 매개변수 이름은 보통 한 문자로 표현한다.
   - 임의의 타입 `T`, 컬렉션 원소 `E`, 맵의 키와 값 `K` / `V`, 예외 `X`, 메서드 반환 타입 `R`을 주로 사용한다.

## 문법 규칙 (유연하지만 문맥을 명확히 하는 형태)
7. 클래스와 인터페이스 (객체 생성 여부에 따라 다름)
   - 객체를 생성할 수 있는 일반 클래스는 단수 명사나 명사구를 사용한다 (예: `Thread`, `PriorityQueue`).
   - 객체를 생성할 수 없는 유틸리티 클래스 등은 복수형 명사로 짓는다 (예: `Collections`).
   - 인터페이스는 클래스와 똑같이 짓거나, able 혹은 ible로 끝나는 형용사로 짓는다 (예: `Runnable`, `Accessible`).

8. 메서드 (반환값과 역할에 따라 다름)
   - 어떤 동작을 수행한다면 동사나 동사구로 짓는다 (예: append, drawImage).
   - `boolean` 값을 반환한다면 보통 is나 has로 시작한다 (예: isEmpty, hasSiblings).
   - 인스턴스의 속성을 반환한다면 명사, 명사구, 혹은 `get`으로 시작하는 동사구로 짓는다 (예: size, getTime).

**특별한 메서드 명명 규칙**
    - 다른 타입의 객체로 변환하여 반환: `toType` 형태 (예: toString, toArray).
    - 다른 뷰(View)로 보여주는 객체를 반환: `asType` 형태 (예: asList).
    - 기본 타입 값으로 반환: `typeValue` 형태 (예: intValue).
    - 정적 팩터리 메서드: `from`, `of`, `valueOf`, `instance`, `getInstance`, `newInstance` 등을 흔히 사용한다. ()
