# 다른 타입이 적절하다면 문자열 사용을 피하라

### 1. 문자열은 다른 값 타입을 대신하기에 적합하지 않다.

모든 타입은 문자열로 표현할 수 있지만, 문자열이 모든 타입을 표현할 수 있는 것은 아니다.
문자열은 다른 타입을 대신하기에 적합하지 않다. 

예를 들어, 날짜를 문자열로 표현하는 것은 좋지 않다.
날짜는 연도, 월, 일과 같은 여러 구성 요소로 이루어져 있으며, 문자열로 표현하면 이러한 구성 요소를 명확하게 구분하기 어렵다.

또한, 문자열은 날짜의 유효성을 검사하기 어렵게 만들며, 날짜 연산을 수행하는 것도 복잡하게 만든다. 
따라서, 날짜와 같은 복잡한 데이터를 표현할 때는 문자열 대신 적절한 데이터 타입을 사용하는 것이 좋다.

위 예시와 같은 문제가 발생하기에 대체할 수 있는 타입이 존재한다면 문자열을 사용하지 않는 것이 좋다.

만약 대체할 수 있는 타입이 없다면 VO(Value Object)를 만들어서 문자열을 감싸는 것이 좋다.

### 2.문자열은 혼합 타입을 대신하기에 적합하지 않다.

여러 요소가 혼합된 데이터를 하나의 문자열로 표현하는 것은 좋지 않다.

에를들어 아래와 같은 혼합 타입은 혼란을 야기한다.

```java
String compoundKey  = className + "#" + i.next(); 
```

`className`과 `i.next()` 서로 다른 타입을 `#` 으로 구분하고 있다.
만약 `className` 에 `#` 문자가 포함되어 있다면 어느 부분을 기준으로 구분해야 하는지 알 수 없게되는 문제가 발생한다.
또한, 문자열 파싱은 성능도 좋지 않고 적절한 비교 메서드(`equal`, `compareTo`...)를 제공할 수 없다.

### 3. 문자열은 권한을 표현하기에 적합하지 않다.

클라이언트가 제시하는 문자열 키를 기반으로 `ThreadLocal`를 사용하여 권한을 표현하는 것은 좋지 않다.

```java
public class ThreadLocal {
    private ThreadLocal() {}
    
    static final Map<String, Object> map = new HashMap<>();

    public static void set (String key, Object value) {
        //...
    }
    
    public static Object get(String key) {
        //...
    }
}
```

해당 코드는 스레드 구분을 위해 문자열 키를 사용한다.

하지만 이러한 문자열 키는 중복된 키가 존재할 수 있고, 오타로 인해 잘못된 키가 사용될 수 있다.
이러한 경우 동일한 스레드를 공유하게 되고 이는 악의적으로 사용되거나 정보가 유실될 수 있는 보안 취약점이 될 수 있다.

```java
public class ThreadLocal {
	private ThreadLocal() {}

	static final Map<Key, Object> map = new HashMap<>();

	public static void set (Key key, Object value) {
		//...
	}

	public static Object getKey() {
		return new Key();
	}
}
```

위와 같이 문자열 대신 `Key` 클래스를 만들어서 사용하는 것이 좋다.
이를 통해 클라이언트는 고유한 `Key` 인스턴스를 사용해 권한을 표현할 수 있고, 중복된 키나 오타로 인한 문제를 방지할 수 있다.

하지 현재 코드도 개선할 수 있는 부분이있다.

- Object 타입 대신 제네릭을 사용하여 타입 안전성을 높이는 것이 좋다. 

### 최종 개선한 Th
```java
public class Thread {
    ThreadLocalMap threadLocals = null; 
}

public final class ThreadLocal<T> {
	public ThreadLocal() {};
    public void set(T value) {
        Thread currentThread = Thread.currentThread(); // 현재 내 스레드를 찾아서
        Map map = currentThread.threadLocals;          // 내 스레드의 개인 주머니를 열고
        
        // 나 자신(this == ThreadLocal 인스턴스)을 키로 삼아서 저장!
        map.put(this, value); 
    }
	public T get();
}
```

메모리 누수 방지를 위해 HashMap 대신 WeakHashMap을 사용한다.

## 결론

더 적합한 데이터 타입이 있거나 새로 작성할 수 있다면, 문자열 대신 새로운 타입을 사용하자
