# 38 - 확장할 수 있는 열거 타입이 필요하면 인터페이스를 사용하라
핵심 : Enum은 원래 상속이 안 되지만, 인터페이스를 쓰면 상속처럼 보이게 만들 수 있다.

- 사실 대부분 상황에서 열거 타입을 확장하는 건 좋지 않은 생각이다. 라고 책에서 말하고 있다.
  - 어떤 이유 때문일까?
    - Enum의 가장 큰 장점 중 하나인 `values()`로 전체를 순회하는 기능이 깨지기 때문이다.

하지만 확장할 수 있는 열거 타입이 어울리는 쓰임이 최소한 한가지 있다고 한다.

예를 들어 Operation 인터페이스를 구현하는 열거 타입이 있다고 하자.

이때 Enum 타입을 활용해 Operation 인터페이스를 구현 하는 열거 타입을 만들 수 있다.

```java
public interface Operation {
    double apply(double x, double y);
}
```

```java
public enum BasicOperation implements Operation {
    PLUS("+") {
        public double apply(double x, double y) { return x + y; }
    },
    MINUS("-") {
        public double apply(double x, double y) { return x - y; }
    },
    TIMES("*") {
        public double apply(double x, double y) { return x * y; }
    },
    DIVIDE("/") {
        public double apply(double x, double y) { return x / y; }
    };

    private final String symbol;

    BasicOperation(String symbol) {
        this.symbol = symbol;
    }

    @Override
    public String toString() {
        return symbol;
    }
}
```
item 34 에서 설명한 것처럼 열거 타입은 상수별로 다르게 동작하는 코드를 구현할 수 있다.
이 때 추상 메서드를 선언하고, 각 상수에서 이를 구현하는 방식도 있지만 인터페이스를 구현하는 열거 타입을 만드는 방식도 있다.

이때 추상 메서드가 아닌 인터페이스를 구현하는 방식을 사용하면 다형성을 활용할 수 있다는 장점이 있다.

예를 들어 `BasicOperation` 외에 `Operation` 인터페이스를 구현하는 또 다른 열거 타입을 만들 수 있다.

```java
public enum ExtendedOperation implements Operation {
    POWER("^") {
        public double apply(double x, double y) { return Math.pow(x, y); }
    },
    REMAINDER("%") {
        public double apply(double x, double y) { return x % y; }
    };

    private final String symbol;

    ExtendedOperation(String symbol) {
        this.symbol = symbol;
    }

    @Override
    public String toString() {
        return symbol;
    }
}
```

### 결론
열거 타입 자체는 확장할 수 없지만, 인터페이스와 그 인터페이스를 구현하는 기본 열거 타입을 함께 사용하면 확장의 효과를 낼 수 있다.
이를 통해 클라이언트는 기존 연산을 그대로 사용하면서 새로운 연산을 추가할 수 있게 된다.
